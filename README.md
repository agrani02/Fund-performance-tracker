# Fund Performance Tracker

NAV ranking and AUM tracking dashboard for Capitalmind Mutual Fund — compares CM funds against peers across 4 categories.

## Dashboard Tabs

| Tab | Funds | Returns |
|-----|-------|---------|
| **Liquid Ranking** | 42 funds | Annualised — 1D/7D/30D/91D/184D/365D/SI |
| **ARB Ranking** | 37 funds | Annualised — 1D/7D/30D/91D/184D/365D/SI |
| **Multi Asset Ranking** | 34 funds | **Absolute** — 1D/7D/30D/91D/184D/365D/SI |
| **Flexi Cap Ranking** | 45 funds | **Absolute** — 1D/7D/30D/91D/184D/365D/SI |
| **Liquid AUM** | 42 funds | AUM history — 7D/30D % and absolute change |
| **ARB AUM** | 37 funds | AUM history — 7D/30D % and absolute change |
| **Multi Asset AUM** | 34 funds | AUM history — 7D/30D % and absolute change |
| **Flexi Cap AUM** | 45 funds | AUM history — 7D/30D % and absolute change |

> Liquid and ARB returns are annualised `(NAV_t/NAV_p − 1) × (365/n) × 100` — comparable to repo rate and other debt benchmarks.
> Multi Asset and Flexi Cap returns are absolute `(NAV_t/NAV_p − 1) × 100` — SEBI mandates absolute for equity funds under 1 year.

## Tech Stack

- **Backend:** Python 3 / Flask (`fpt_app_full.py`) — port 8082
- **Frontend:** Single-file HTML/JS (`fpt_dashboard_full.html`)
- **Data:** SQLite databases, AMFI India NAV history portal, AMFI fundperformance API
- **Excel export:** xlsx-js-style (client-side, all 4 ranking sheets)

## Return Formula

**Liquid & ARB (backfill):**
```
return = (NAV_today / NAV_prev − 1) × (365 / n_days) × 100
```

**Multi Asset & Flexi Cap (backfill):**
```
return = (NAV_today / NAV_prev − 1) × 100
```

**Daily scripts** pull pre-calculated returns from the AMFI fundperformance API.
Multi Asset and Flexi Cap daily scripts de-annualise AMFI's figures: `amfi_val × n_days / 365`.
1D return uses `preNavDirect / navDirect` directly from the API response.

## API Endpoints

```
GET  /api/nav/dates                   — Liquid NAV dates (+ full_count_date: most recent date with 42 funds)
GET  /api/nav/rankings?date=          — Liquid fund rankings for a date
GET  /api/nav/aum-tracker             — Full liquid AUM history + 7D/30D change
POST /api/nav/override                — Re-run liquid ingest with custom T-1 date

GET  /api/arb/dates                   — ARB NAV dates (+ full_count_date: most recent with 37 funds)
GET  /api/arb/rankings?date=          — ARB fund rankings for a date
```

## Dashboard Features

- **Full-count date default** — each tab auto-selects the most recent date where the full peer universe published NAV (e.g. if only 33/34 Multi Asset funds published today, shows yesterday's full 34-fund data)
- **NAV Date column** — shows the actual date each fund's data was fetched
- **NAV DATE toggle** — global show/hide for the NAV Date column across all tabs
- **T-1 Override** — per-tab date picker to reprocess a date with a custom previous-day NAV (for holidays)
- **Sort** — click any period column header or use the sort dropdown
- **Excel export** — downloads all 4 ranking sheets with heatmap colouring; filename uses NAV date
- **Missing fund detection** — daily scripts print `⚠ MISSING` if a fund that appeared yesterday is absent today

## Critical Settings

**AMFI `__gsas` cookie** — used in all 4 daily scripts for the fundperformance API. Refresh from browser DevTools → Network tab on `amfiindia.com` when AUM stops updating.

**Universe sizes** (hardcoded in `fpt_app_full.py` for `full_count_date` logic):
- Liquid: 42 | ARB: 37 | Multi Asset: 34 | Flexi Cap: 45

## Known Issues / Notes

- Multi Asset and Flexi Cap backfill use 30-day chunks — AMFI rate-limits large history requests; run early morning
- Some date ranges may return 0 rows from AMFI history — re-run with `--from YYYY-MM-DD`
- Old Bridge is excluded from ARB AUM display
- AMFI doesn't publish 7D/30D returns for funds under ~7/30 days old — those cells show `—`
- Multi Asset and Flexi Cap DBs must be wiped and rebuilt if return formula changes (absolute vs annualised)
